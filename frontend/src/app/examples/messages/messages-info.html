General info

This is an example page to demonstrate how this boilerplate handles errors with $http and $sailsSocket requests. Note that this magic is done automatically via the error interceptor so you don't have to do anything extra for error handling. The error handling is done via the error interceptor that is attached to $http and $sailsSocket services.

This interceptor will catch all errors from $http and $sailsSocket requests and show those to the user via the message service. Note that the message shown depends on the actual error response from the backend.

Functions in this example

Custom messages
Simple example to trigger different types of messages: info, success, warning and error with specified title and actual message.
$http / $sailsSocket
Examples to demonstrate invalid URL and not found record. These are handled automatically by ErrorInterceptor.
